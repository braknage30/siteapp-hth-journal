# siteapp-hth-journal

本仓库用于归档和发布多个独立的 HTML 页面。这些页面通常对应不同功能模块或展示内容，不依附于特定域名或具体网站。

## 项目简介

`siteapp-hth-journal` 是一个静态 HTML 页面集合仓库。其主要目的是保存和整理独立编写的页面源码，便于版本管理、历史回溯以及通过 GitHub Pages 等方式进行简单的公开访问。

## 目录说明

```
├── index.html          # 首页或导航页（可选）
├── page1/              # 页面功能模块一
│   ├── index.html
│   └── assets/         # 该页面使用的资源文件
├── page2/              # 页面功能模块二
│   ├── index.html
│   └── assets/
└── README.md           # 本文件
```

- 每个独立页面通常放置在一个以功能或内容命名的子目录中。
- 资源文件（如 CSS、JavaScript、图片等）建议存放在对应页面的 `assets` 文件夹内。
- 根目录下可放置 `index.html` 作为总入口或导航页。

## 页面归档说明

- 所有页面均为静态 HTML，不依赖后端服务。
- 页面之间相互独立，不共享数据或状态。
- 归档内容可能包含不同时期、不同用途的页面，格式和风格可能不一致。
- 部分页面可能包含外部引用（如 CDN 资源、字体库等），请确保网络环境可正常加载。

## 维护说明

- 本仓库主要供个人或小团队归档使用，不保证所有页面持续兼容最新浏览器。
- 欢迎提交 Issue 或 Pull Request 来修正明显错误或改进页面内容。
- 新增页面时，请尽量保持目录结构清晰，并在根目录 README 中简要说明新增内容。
- 不建议在本仓库中存放敏感信息或大体积二进制文件。

## 使用方式

你可以直接克隆本仓库到本地，通过浏览器打开任意 HTML 文件即可查看对应页面。若希望在线访问，可启用 GitHub Pages 功能（通常指向 `docs/` 目录或根目录）。
